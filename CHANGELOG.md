# История изменений

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/),
версии — по [семантическому версионированию](https://semver.org/lang/ru/).

## [Не выпущено]

### Добавлено
- `create_supplier_invoice` — регистрация входящего «Счёта на оплату поставщика»
  (основание для оплаты, в т.ч. предоплаты до поставки): без проводок и счетов
  учёта, единая ТЧ «Товары» под товары и услуги (поле `content`/Содержание для
  доставки), реквизиты входящего счёта (`incomingNumber`/`incomingDate`).
- `create_folder` и `move_to_folder` — папки/группы иерархических справочников
  (по умолчанию Контрагенты), вложенность по имени или Ref_Key.
- В `create_nomenclature` — артикул, размещение в папке (`folder`) и признак
  услуги (`isService`).
- В `create_contract` — руководитель/подписант контрагента (`headName`,
  `headPosition`).

### Изменено
- `create_contract`: вид договора (`kind`) теперь обязателен — коннектор не
  угадывает покупатель/поставщик; номер по умолчанию «б/н», дата — сегодня.

## [0.1.3] — 2026-06-16

### Добавлено
- Юнит-тесты (Vitest, без живой базы) на ключевую логику: билдер запросов и
  регресс кодирования `+`/`%20`, разбор конфигурации (мультибаза), классификация
  метаданных, ошибки, фоллбэк публикации.
- CI на GitHub Actions: проверки на push/PR и публикация в npm по тегу через
  **Trusted Publishing** (OIDC, без токена; provenance автоматически).
- Аннотации инструментов MCP (`readOnlyHint` / `destructiveHint` / `openWorldHint`)
  и `instructions` сервера.
- ESLint + Prettier; `CONTRIBUTING.md`, `SECURITY.md`, шаблоны issue, Dependabot;
  `CHANGELOG.md`.

### Изменено
- Документация разведена: `README.md` — про проект (возможности, примеры,
  ограничения), `docs/CONNECTING.md` — установка/настройка/диагностика. Добавлены
  badges и SEO-формулировки, вариант подключения через `npx`.

### Прочее
- SEO: расширены `keywords` в `package.json` и добавлены topics репозитория.

## [0.1.2] — 2026-06-16

### Добавлено
- «Богатая» карточка контрагента: телефон, email, адрес (табличная часть
  «Контактная информация») и ОГРН (если в базе настроен доп.реквизит) в
  `create_counterparty` / `update_counterparty`.
- `create_bank_account` — расчётный счёт контрагента (банк по БИК), опция «основной».
- `create_contact_person` — контактное лицо контрагента (директор и т.п.), опция «основное».

## [0.1.1] — 2026-06-16

### Исправлено
- Формулировки в документации: убрано устаревшее «только на чтение» как природа
  сервера (запись доступна через гейт); оставлено корректное «read-only по умолчанию».

## [0.1.0] — 2026-06-16

Первый публичный релиз в npm.

### Добавлено
- **Чтение и аналитика:** список баз/организаций, карта объектов и поля
  (`list_entities` / `describe_entity`), поиск и карточка контрагента, поиск и
  получение документов, дебиторка (сч. 62), остатки (сч. 41/10/43), продажи,
  движение денег, история взаиморасчётов, `health_check`.
- **Мультибаза и мультиорганизация:** параметры `database` и `organization`.
- **Запись (через двойной гейт `READ_ONLY` + `WRITABLE` и dry-run):** создание
  контрагентов, номенклатуры, договоров (вид/валюта/тип цен), счетов покупателю,
  поступлений и реализаций, актов услуг, оплат; проведение/отмена (`post_document`);
  редактирование строк документов; изменение реквизитов (`update_*`); мягкое
  удаление (`mark_for_deletion`).
- Счета учёта документов берутся из регистра «Счета учёта номенклатуры» (откат на
  коды плана счетов); вежливый фоллбэк при отсутствии объекта в «Составе OData».
- npm-пакет с запуском через `npx 1c-odata-mcp`.
