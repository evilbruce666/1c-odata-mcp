# Как внести вклад

Спасибо за интерес к проекту! Issue и pull request приветствуются.

## Разработка

Нужен **Node.js ≥ 20**.

```bash
git clone https://github.com/evilbruce666/1c-odata-mcp.git
cd 1c-odata-mcp
npm install
npm run build        # сборка TypeScript → dist
npm test             # юнит-тесты (без живой базы)
npm run lint         # ESLint
npm run format       # Prettier (форматирование)
npm run typecheck    # проверка типов
```

Перед PR прогоните локально то же, что и CI: `lint`, `format:check`, `typecheck`,
`build`, `test` — иначе CI не пройдёт.

## Тесты

- **Юнит-тесты** (`test/`, Vitest) проверяют чистую логику без обращения к 1С —
  билдер запросов, разбор конфигурации, классификацию метаданных, ошибки. Их
  достаточно для большинства правок.
- **Проверка на живой базе** делается вручную read-only probe-скриптами
  (`dist/scripts/probe-*.js`, запуск `node --env-file=.env …`) — секретов они не
  печатают. Для пишущих изменений тестируйте на **тестовой/копии** базы.

## Стиль и коммиты

- Код форматируется Prettier и проверяется ESLint (в CI).
- Комментарии и сообщения для пользователя — на русском (целевая аудитория).
- Коммиты — в стиле Conventional Commits (`feat:`, `fix:`, `docs:`, `chore:`, `ci:` …).

## Идеи для вклада

- Поддержка других конфигураций 1С (УТ, КА, ERP) в аналитике.
- Дополнительные документы и справочники для записи.
- Тесты на новые модули.
