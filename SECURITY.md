# Политика безопасности

## Сообщить об уязвимости

Пожалуйста, **не открывайте публичный issue** для уязвимостей.

Сообщайте приватно через GitHub: вкладка **Security → Report a vulnerability**
(приватные Security Advisories). Так детали не станут публичными до выпуска
исправления.

Опишите проблему, шаги воспроизведения и потенциальное влияние. Мы постараемся
ответить в разумный срок.

## Что стоит знать о безопасности проекта

- Сервер по умолчанию работает **только на чтение**; запись включается двумя
  независимыми предохранителями (`READ_ONLY=false` + `ODATA_DB_<ИМЯ>_WRITABLE=true`)
  и работает через предпросмотр/подтверждение.
- Секреты (логин/пароль к OData) задаются через переменные окружения; пароль
  редактируется в логах и не попадает в репозиторий.
- Пользовательский ввод не подставляется в `$filter` напрямую — только через
  экранирующий билдер запросов.
- Рекомендуется отдельный пользователь 1С с минимально необходимыми правами и
  ограниченным «Составом OData».
